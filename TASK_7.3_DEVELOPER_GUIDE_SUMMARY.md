## 🚀 使用指南

### 对于新贡献者
1. 阅读完整的 `CONTRIBUTING.md`
2. 按照"开发环境搭建"章节设置环境
3. 查看"代码规范"和"测试规范"
4. 选择一个 good-first-issue 开始贡献

### 对于维护者
1. 在 PR review 中引用相关规范
2. 确保新代码符合文档中的标准
3. 定期更新文档以反映项目变化

### 快速命令参考

```bash
# 环境设置
task dev:setup

# 开发检查
task dev:check

# 运行测试
task backend:test
task backend:test:unit

# 代码质量
task backend:lint
task backend:fmt

# 前端开发
task frontend:dev
task frontend:lint
```

4. **保持测试覆盖率 ≥80%**
5. **及时响应 code review 意见**
